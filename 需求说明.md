项目需求说明书：Telegram Bot 接入酒馆（SillyTavern）
1. 项目背景

目标：让用户通过 Telegram Bot 操作并游玩酒馆，而无需停留在网页端。

约束：酒馆的前端逻辑承载了大量核心功能，直接使用后端 API 限制较大。

方案：使用 Headless Chromium 驱动酒馆前端，Bot 充当“传话人”，实现功能交互。

2. 技术要求

开发语言：Golang

前端模拟：Headless Chromium（如 chromedp / rod）

用户交互：Telegram Bot API

返回结果：非流式，一次性返回完整对话

3. 核心功能

角色卡选择

提供「角色列表按钮」

用户点击按钮 → 切换酒馆前端的角色

聊天记录选择

提供「历史记录按钮分页」

用户选择记录 → Bot 加载对应酒馆聊天记录

聊天对话管理

修改消息：在某条对话后面附带「修改」按钮，触发编辑逻辑

重新生成：在消息后附带「重生」按钮，调用酒馆前端功能重新生成该条对话

补全预设选择

提供「预设列表按钮」

用户点击即可切换酒馆中的补全预设

富文本支持

酒馆前端返回的 HTML 富文本需转义/转换 → 发送到 Telegram

支持基础格式（粗体、斜体、超链接、换行），避免丢失信息

4. 可选功能（低优先级）

编辑补全预设

编辑世界书

编辑角色卡

5. 用户交互设计

尽量按钮化：所有主要操作通过按钮触发，减少命令输入

对话上下文按钮：每条消息下方可附带操作按钮（如「修改」「重生」）

主菜单：

「选择角色」

「选择历史记录」

「选择补全预设」

6. 交付物

Golang 完整代码（含 Bot 主程序 + Headless Chromium 驱动逻辑）

Dockerfile（可选，便于部署）

README（说明配置、运行步骤、依赖环境）
